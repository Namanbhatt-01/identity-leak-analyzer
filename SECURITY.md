# Security Policy

## Supported Versions

We actively support and maintain the latest version of the framework. Because this repository operates as an offline simulation pipeline, no remote APIs or network listeners are exposed.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

We take the security and integrity of this project seriously. If you find a security vulnerability, please do **not** open a public GitHub issue. Instead, report it privately:

1. Send an email to **namanbhatt73@gmail.com** containing detailed description, proof of concept (PoC), and steps to reproduce.
2. We will acknowledge your report within 48 hours.
3. A patch or mitigation will be developed and merged.

Thank you for helping keep this project secure!
